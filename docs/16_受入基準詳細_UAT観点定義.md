# 受入基準詳細（UAT観点定義）

## 1. 目的
- `12_UATチェックリスト` の各項目に合否基準を付与し、判定のばらつきをなくす。

## 2. 判定ルール
- 合格: 期待結果を満たし、重大障害（業務停止/データ破壊）がない
- 条件付き合格: 軽微不具合あり、回避手順が確立
- 不合格: 期待結果未達または重大障害あり

## 3. 観点別受入基準
### 3.1 認証
- ログイン成功: 正しいID/PWで `/dashboard` 遷移
- ログイン失敗ロック: 5回失敗後に423応答
- 未ログインアクセス: 業務画面は `/login` へリダイレクト

### 3.2 資産管理
- 資産登録: 必須項目で登録可能、監査ログ記録あり
- PC利用者必須: PC種別で `person_attr_01` 未設定はエラー
- 予算後付け: budget-link更新で状態がLINKEDへ遷移

### 3.3 構成/予算/ライセンス
- 構成登録: 廃棄済資産を含まない場合成功
- 廃棄済資産追加拒否: 422で拒否
- 執行済予算登録: 予算ID指定で作成成功
- ライセンス上限: total超過割当が422で拒否

### 3.4 PC/棚卸
- PC割当: PC資産のみ登録可
- 棚卸結果: 登録/更新が重複制約を満たして成功

### 3.5 CSV/バッチ
- CSV出力: ヘッダ含むCSV取得
- CSV取込: 正常行が取り込まれ、エラー行は明細返却
- バッチ実行: 4コマンドが例外なく終了

### 3.6 監査
- 主要更新操作に監査ログが残る
- 監査ログ画面でフィルタ閲覧できる
